분석 역량 정의

01분석 기획Planning

분석 주제에 맞는 데이터를 고르고, 결과를 보면서 다음 질문을 계속 이어간다. 어떤 데이터로 어떤 순서로 질문할지 처음부터 끝까지 스스로 설계할 수 있다.

02원인 규명Diagnosis

숫자가 이상할 때, 수집부터 적재까지 데이터가 만들어지는 과정을 거슬러 올라가며 어디서 왜 틀어졌는지 찾아낸다. 표면 원인에서 멈추지 않고 근본까지 파고든다.

03상황 해석 · 의사결정Insight

분석 결과가 지금 무엇을 말하는지, 앞으로 어떻게 될지 읽어내고 어떤 결정을 해야 하는지까지 짚는다. 숫자에서 결정으로 스스로 이어갈 수 있다.

레벨 기준 — 도구 숙련도

GA4수집 플랫폼
Lv 1인식 전이름은 알지만 직접 열어본 적 없음. 어떤 데이터가 수집되는지 관심 자체 없음
Lv 2기초대시보드에서 PV, 세션 수 읽는 수준. 왜 그 숫자가 나오는지, 어디서 오는지 모름
Lv 3독립 수행날짜 필터, 세그먼트, 기본 리포트를 스스로 조작함. 이벤트를 이해하고 원하는 기준으로 리포트를 직접 구성함
Lv 4주도·설계이벤트 수집 구조를 이해함. GA4 → BigQuery 적재 흐름 파악, 수집 설계 자체에 의견을 낼 수 있음
BigQuery분석 도구
Lv 1인식 전접근 권한도 없고 어떻게 여는지 모름. GA4 데이터가 여기 쌓인다는 것 자체를 몰랐음
Lv 2기초기본 SELECT · WHERE · GROUP BY 작성 가능. 누군가 짜놓은 쿼리를 조건만 바꿔서 쓰는 수준
Lv 3독립 수행JOIN, 서브쿼리, 윈도우 함수를 스스로 조합해서 분석 쿼리를 처음부터 설계함. 테이블 구조 파악하고 직접 꺼냄
Lv 4주도·설계파티션 테이블 구조 이해, 비용·속도 고려해서 쿼리 최적화. 스케줄 쿼리로 자동화하고 대용량 처리 전략을 스스로 세움
Python분석 도구
Lv 1인식 전코드 복붙 수준. 오류 메시지가 뜨면 무슨 뜻인지 몰라서 거기서 멈춤
Lv 2기초pandas로 기본 데이터 조작 가능. 주어진 코드 구조 안에서 조건이나 컬럼 정도 바꿀 수 있음
Lv 3독립 수행데이터 전처리, 집계, 시각화를 처음부터 설계하고 구현함. 분석 목적에 맞게 코드 구조를 스스로 짬
Lv 4주도·설계외부 플랫폼 API와 직접 연결해서 데이터를 자동으로 수집·적재하는 파이프라인을 구성함. 반복 작업을 자동화하고 재사용 가능한 구조로 정리해둠
Salesforce 연동수집 플랫폼
Lv 1인식 전분석에 어떻게 쓰는지 모름. Python이나 DBeaver로 직접 데이터를 꺼낼 수 있다는 생각 자체 없었음
Lv 2기초회원 데이터가 Salesforce에 있다는 건 알고 있음. 직접 접근해본 적 없고 필요하면 누군가한테 부탁하는 방식
Lv 3독립 수행DBeaver 또는 API 연동 설정을 스스로 완료함. 원하는 회원 데이터를 직접 꺼내서 분석에 씀
Lv 4주도·설계어떤 데이터가 어떤 구조로 쌓여 있는지 파악하고 필요한 데이터를 스스로 설계해서 추출함. Python · DBeaver와 연동한 분석 흐름 전체를 직접 구성함
DBeaver분석 도구
Lv 1인식 전설치해본 적 없음. DB에 직접 붙어서 데이터를 꺼낸다는 개념 자체 없음. 파일로 받거나 누군가 뽑아주길 기다림
Lv 2기초연결 설정은 도움을 받아 완료한 상태. 기본 SELECT 쿼리로 단순 조회 가능
Lv 3독립 수행DB 연결 설정을 스스로 완료함. 여러 테이블을 JOIN하고 복잡한 조건의 쿼리를 직접 작성해서 원하는 데이터를 꺼낼 수 있음
Lv 4주도·설계BigQuery, Salesforce 등 여러 DB를 동시에 연결해서 운영함. 쿼리 구조를 최적화하고 반복 추출 작업을 쿼리 단에서 정리해둠

레벨 기준 — 분석 역량

분석 기획Planning
Lv 1인식 전시키는 것만 함. 다음 질문을 만들어야 한다는 개념 자체 없음. 결과가 나오면 거기서 끝
Lv 2기초질문이 주어지면 데이터는 꺼냄. 그 결과를 보고 뭘 더 봐야 하는지 스스로 판단 못하고 매번 다음 스텝을 물어봄
Lv 3독립 수행결과를 보고 후속 질문을 스스로 만들어냄. 다만 분석 전체 흐름을 처음부터 설계하는 건 어려움
Lv 4주도·설계어떤 데이터로 어떤 질문을 어떤 순서로 던질지 스스로 설계하고 끝까지 이어감. 분석의 시작과 끝을 혼자 그릴 수 있음
원인 규명Diagnosis
Lv 1인식 전숫자가 이상해도 그냥 넘기거나 단순 보고로 끝냄. 파이프라인을 거슬러 올라가야 한다는 발상 자체 없음
Lv 2기초이상하다고 보고는 하지만 직접 파고들지 않음. 원인 파악은 누군가 해주길 기다림
Lv 3독립 수행데이터를 직접 역추적해서 표면적 원인을 찾아냄. 다만 거기서 멈추고 더 근본적인 원인까지 나아가지 못함
Lv 4주도·설계수집부터 적재까지 전 과정을 역추적해서 어디서 왜 틀어졌는지 스스로 찾아냄. 표면 원인에서 멈추지 않고 근본 원인까지 파고듦
상황 해석 · 의사결정Insight
Lv 1인식 전숫자를 꺼내서 전달하는 게 분석의 끝이라고 생각함. 의미를 연결하려는 시도 없음
Lv 2기초결과를 전달하긴 하지만 숫자 나열 수준. 왜 이런 결과가 나왔는지, 좋은 건지 나쁜 건지 설명 못함
Lv 3독립 수행현재 상태가 어떤 의미인지 설명 가능. 앞으로 어떻게 될지, 어떤 결정을 해야 하는지까지는 연결 못함
Lv 4주도·설계분석 결과로 지금 상황을 진단하고, 앞으로 어떻게 될지 판단해서 어떤 결정을 해야 하는지까지 짚어냄. 숫자에서 결정까지 스스로 이어감
